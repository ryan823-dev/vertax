## 🔴 P0 - 正式上线前必须完成

### 1. 生产环境可用性验收
- [ ] **完成关键第三方配置核查**：逐项确认生产环境中的 DashScope、Resend、OSS、CRON_SECRET、Twitter/X、Facebook、YouTube 等配置均为真实可用值，而非占位符。
- [ ] **完成生产账号授权实测**：对 Social 模块中已接入的平台做真实账号联调，确认 OAuth / API Keys、token 刷新、发布权限均正常。
- [ ] **确认数据库与对象存储一致性**：核对生产环境 PostgreSQL、Prisma schema、OSS bucket、CORS 与上传回调行为一致，避免“本地可用、线上异常”。

### 2. 端到端业务闭环验收
- [ ] **跑通知识库 → 获客 → 内容 → 发布主链路**：至少完成一轮人工验收，从资料上传、企业画像/品牌资料沉淀，到雷达发现、内容生成、社媒发布，确认核心价值链路真实闭环。
- [ ] **验证 AI 流式交互稳定性**：对关键 Skill 的 SSE 流式输出进行手工验收，确认长耗时任务在真实网络下无中断、无假完成、无前端卡死。
- [ ] **抽检知识库导入质量**：对站点导入后的页面样本进行人工抽检，确认当前规则过滤后，低价值页面不会大规模污染知识库。

### 3. 上线前测试与回归
- [ ] **执行完整 E2E 回归测试**：运行当前 Playwright 核心用例，确认导航、雷达、知识引擎、增长系统等关键页面在待发布版本上全部通过。
- [ ] **执行构建与基础质量检查**：至少完成 `build`、Prisma generate、关键路由访问检查，避免出现“代码可读但版本不可发”的问题。

## 🟡 P1 - 建议在首发窗口内补齐

### 1. 获客雷达 (Radar)
- [ ] **可视化 AI 理由**：在候选详情中展示 `matchReasons`、`approachAngle` 等字段，增强用户对 AI 评分和推荐的信任。
- [ ] **线索价值沉淀**：为 Outreach Pack 增加“保存到草稿/内容库”能力，避免生成结果只停留在瞬时页面中。
- [ ] **排除逻辑回流**：将人工排除的候选特征沉淀回搜索画像/排除规则，形成越用越准的筛选闭环。

### 2. 增长系统 (Marketing)
- [ ] **统一创作入口**：收敛 Marketing 内多套内容创建入口，避免用户在“首页快速创建”和“正式 briefs 流程”之间混淆。
- [ ] **GEO 自动分发闭环**：将内容生成后的注册/分发动作进一步自动化，减少手工点击。
- [ ] **编辑器上下文保护**：对依赖当前文稿上下文的 Skill 增加禁用态或校验提示，避免空调用。

### 3. 社交媒体 (Social)
- [ ] **跨模块内容引用**：支持从内容库直接选取文章并自动适配为多平台社媒版本。
- [ ] **LinkedIn 发布策略升级**：当前为 Share URL 模式，可评估是否补齐更深度的平台直连发布能力。

### 4. 知识引擎 (Knowledge)
- [ ] **页面质量评分增强**：在现有 URL 规则过滤之外，增加页面语义质量评估/AI 质检，进一步降低法律页、导航页、无信息页进入知识库的概率。

## 📋 建议的上线门槛（Go-Live Gate）

满足以下条件后，再进入正式外发/市场发布：

1. **生产环境配置完成**：关键外部服务全部通过真实值校验。
2. **端到端主链路跑通**：至少 1 条真实业务路径完整通过。
3. **核心测试通过**：E2E 核心套件、构建、关键路由访问全部正常。
4. **知识库质量可接受**：导入样本抽检结果满足预期。
5. **对外材料同步更新**：README、用户手册、销售演示材料与当前产品一致。
