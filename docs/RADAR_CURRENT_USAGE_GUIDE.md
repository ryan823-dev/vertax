# 获客雷达当前版本使用说明

更新时间：2026-04-16

本文基于当前仓库内已经实现的页面、动作和数据结构整理，覆盖当前主入口 `/customer/radar` 及其相关页面，重点说明“现在这个版本实际怎么用”。

## 1. 这套系统现在是怎么工作的

当前版本的获客雷达，主线不是“手工填很多筛选表单”，而是：

1. 先确认系统按什么目标客户画像找客户。
2. 再让系统自动搜索、持续发现候选对象。
3. 对候选做审核、分层、排除。
4. 把值得继续推进的对象导入线索库。
5. 在线索库里继续补全联系人、生成外联材料、推进跟进。
6. 对采购/招投标类机会，单独沉淀到“采购机会”看板。

从产品结构上看，当前版本是一条“画像驱动”的流水线：

- 目标客户画像
- 自动搜索
- 候选池
- 线索库
- 采购机会

另外，当前版本还额外挂了一个日常执行页：

- 今日外联

它的作用不是替代主流程，而是把今天最值得马上联系、补全、推进的对象集中起来，方便销售或运营按日执行。

## 2. 使用前准备

正式使用前，建议先确认 4 件事：

1. 已登录客户侧系统，并进入 `/customer/radar`。
2. 知识引擎里至少已经有基础企业认知。
常见来源包括 `Company Profile`、`ICP Segment`、`Persona`、`Evidence`。
3. 团队已经明确当前想打的行业、区域、产品关键词。
4. 已理解当前版本的核心原则：
系统先按画像找，再由人审核、导入、推进，不建议跳过候选审核直接大规模外联。

如果知识引擎里还没有企业认知内容，雷达首页会把你引到“目标客户画像”页先做同步。

## 3. 当前版本的标准使用路径

### 第一次启动

建议首次使用按下面顺序操作：

1. 进入“目标客户画像”，同步最新画像。
2. 检查 AI 生成的 `TargetingSpec` 和 `ChannelMap` 是否合理。
3. 回到“自动搜索”，先跑一轮即时搜索。
4. 进入“候选池”，把结果分成 A / B / C / 排除。
5. 把 A / B 中值得继续推进的公司导入“线索库”。
6. 在线索库补联系人、生成外联包、开始跟进。
7. 对明确采购或招投标类对象，单独导入“采购机会”。

### 日常运营

建议每天按下面节奏使用：

1. 先看“今日外联”，处理今天最适合联系的对象。
2. 再看“候选池”，消化新增待审核候选。
3. 最后回到“线索库”，继续补全和推进高价值公司。

### 周期性维护

建议每周至少做 1 次：

1. 回到“目标客户画像”同步最新画像。
2. 检查“自动搜索”里当前启用的数据源与搜索配置状态。
3. 清理明显不适合的候选，减少后续噪音。

## 4. 各页面怎么用

## 4.1 总览页 `/customer/radar`

这是当前版本的雷达总入口，主要做 3 件事：

1. 看雷达流水线当前走到哪一步。
2. 直接进入各模块。
3. 用“画像校正助手”补充客户自己的行业判断，校正目标客户画像。

当前页上你会看到：

- 五步流程状态
- 当前统计卡片
- 各模块快捷入口
- 画像校正助手

### 画像校正助手的正确用法

这个入口适合让客户用自然语言补充自己的行业判断。客户不是在当搜索员，而是在校正系统应该怎样理解目标客户画像。

示例：

- 美国汽车零部件工厂，有 paint booth 改造需求
- 欧洲家电工厂，关注喷涂自动化和 VOC 合规
- 中东地区制造企业中，优先看自动化喷漆产线升级需求

当前版本里，这个助手会做两件事：

1. 调用 `/api/radar/parse-request` 把客户的自然语言判断解析成画像校正字段。
2. 创建新的 `TargetingSpec` 版本，把国家、行业、关键词、排除对象、应用场景和触发器沉淀到目标客户画像里。

当前实现的默认行为是：

- 不把客户输入当成自定义搜索表单。
- 客户输入会作为“行业专家判断”进入画像版本。
- 提交成功后，页面会展示本次新增/合并的市场、行业、关键词、排除对象、应用场景和触发器。
- 自动搜索页继续以 `TargetingSpec` 为主，按最新画像启动或重跑。
- 如果只是本轮临时收窄方向，在自动搜索页使用“本轮画像校准”。

所以，这个入口更像“让画像学习客户的行业经验”，而不是只跑一次搜索。

## 4.2 目标客户画像 `/customer/radar/targeting`

这一页是整个雷达的起点。

当前页负责管理两份核心产物：

- `TargetingSpec`
- `ChannelMap`

### 这两份产物分别是什么

`TargetingSpec` 解决的是：
系统按什么客户特征找客户。

它会包含：

- 行业
- 国家 / 区域
- 企业规模
- 技术关键词
- 使用场景
- 触发信号
- 决策链角色
- 排除规则

`ChannelMap` 解决的是：
系统优先从哪些渠道、用什么查询思路去找这些客户。

它会包含：

- 渠道类型
- 渠道优先级
- 查询语句
- 需要观察的信号

### 这一页的标准操作

1. 点击“同步最新画像”。
2. 等系统从知识引擎生成最新 `TargetingSpec` 和 `ChannelMap`。
3. 先看画像是否准确，再看渠道建议是否合理。
4. 如有偏差，直接进入编辑模式修正。
5. 如有多个版本，可打开版本历史回看。

### 当前版本的特点

- 同步来源是知识引擎，不是手工从零填写。
- 同步后支持手工编辑并保存。
- 支持查看版本历史。
- 支持在 `TargetingSpec` 和 `ChannelMap` 两个标签之间切换。
- 若 AI 给出 `openQuestions`，说明当前画像仍有待确认项，建议先补齐再大规模跑搜索。

### 建议重点检查的字段

建议团队至少检查下面几类字段：

- 行业是否太宽或太窄
- 国家 / 区域是否准确
- 关键词是否偏向你真正卖的产品和场景
- 排除规则是否覆盖无效对象
- 渠道优先级是否符合当前市场打法

## 4.3 自动搜索 `/customer/radar/search`

这一页不是传统高级检索页，而是“让系统按画像执行”和“看系统执行得怎么样”。

当前页主要有两类动作：

1. 立即执行一轮搜索。
2. 管理已有持续搜索配置的运行状态。

### 这页最重要的 3 个按钮

`开始自动搜索`
按当前画像立即跑一轮搜索任务。

`按最新画像重新搜索`
用当前最新画像重新生成查询并执行一轮。

`暂停自动搜索 / 继续自动执行`
针对已有 `RadarSearchProfile` 做启停。

### 自动运行机制（重要）

很多人会把“每天自动跑”理解成“每天人工点一次”，但当前版本不是这样。

当前逻辑是：

1. 用户先做一次启动动作。
2. 系统生成或激活一条持续搜索配置 `RadarSearchProfile`。
3. 后台定时任务按调度规则自动执行，不需要用户每天重复点击。

当前版本里，要把系统切到“持续自动跑”状态，仍然依赖已有 `RadarSearchProfile` 的启停：

- 在自动搜索页对已有搜索配置点击“继续自动执行”，恢复已暂停的搜索策略。
- 管理侧或初始化流程创建并启用 `RadarSearchProfile` 后，后台会按调度规则持续扫描。

也就是说：

- “画像校正助手”负责校正长期画像，不直接代表持续搜索已经启动。
- “开始自动搜索”是按最新画像即时执行。
- “暂停 / 继续自动执行”是控制已有后台持续执行。

### 当前默认是怎么自动跑的

当前版本里，持续搜索配置创建后会带有调度规则，默认常见形式是每天一次。

后台会通过定时任务轮询到期的搜索配置，再自动触发扫描执行。

因此从使用层面可以理解为：

- 用户负责“定义要找谁”和“启动策略”。
- 系统负责“按计划自动去找”。

### 需要特别注意的一点

当前版本里：

- “开始自动搜索”更偏向即时任务。
- “画像校正助手”更偏向把客户专家判断沉淀到长期画像。

也就是说，这两个入口不是完全同一种动作。

### 这页可以看什么

当前页会展示：

- 当前画像摘要
- 系统会启用的能力
- 可用数据源数量
- 数据源健康度和官方源数量
- 已有搜索配置的启用 / 暂停状态
- 最近执行任务

### 当前支持的主要能力类型

页面会按能力卡片解释当前系统可能调用的方向，包括：

- 目录 / 行业名录
- 搜索引擎聚合
- 地图 / POI
- 展会与活动
- 招标 / 采购源
- enrichment 能力
- 联系人识别能力

### 这页的正确使用方式

建议按下面顺序使用：

1. 先确认画像摘要是否正确。
2. 再看系统当前会调用哪些能力。
3. 再补充本轮偏好。
当前页支持填写可选偏好，例如重点国家、重点行业、额外关键词、排除对象等。
4. 再点击“开始自动搜索”或“按最新画像重新搜索”。
5. 执行后，回到候选池消化结果。

### 关于“每天自动跑 XX 个目标客户资料”的说明

当前版本支持的是：

- 按计划自动扫描
- 自动发现候选
- 自动去重
- 自动分层 / 自动补全（取决于配置）

但当前版本还不等于：

- 严格保证每天固定产出 20 / 50 / 100 个新目标客户资料

因为实际产出会受到这些因素影响：

- 当前画像是否准确
- 数据源当天是否有新增
- 去重后剩多少
- 排除规则过滤掉多少
- 自动分层后真正值得保留的有多少

所以更准确的说法应该是：

- 当前版本能实现“启动后按天自动跑”
- 但还不能承诺“每天固定跑出 XX 个有效目标客户资料”

如果后续要做到“每天稳定交付 XX 个目标客户资料”，就需要额外补一层：

- 每日目标数量配置
- 不足时自动补量逻辑
- 结果质量门槛
- 达标 / 未达标反馈机制

## 4.4 候选池 `/customer/radar/candidates`

候选池是雷达里的审核工作台。

这里沉淀的是系统发现但还没有正式进入线索库的对象。

### 当前页可做的事

- 按状态筛选
- 按层级筛选
- 按关键词搜索
- 查看候选详情
- 单个候选分级
- 批量分级
- 批量丰富情报
- 批量导入线索库
- 单个导入为线索
- 单个导入为采购机会
- 排除候选并记录排除原因

### 当前版本的审核逻辑

常用状态包括：

- `NEW`：待审核
- `QUALIFIED`：已分层
- `IMPORTED`：已导入
- `EXCLUDED`：已排除
- `ENRICHING`：补全中

常用层级包括：

- `A`：高匹配、高优先级
- `B`：匹配较清晰，但仍值得补信息
- `C`：相关性偏弱，适合观察
- `excluded`：明确不适合

### 建议怎么审核

1. 先优先看 `NEW`。
2. 把明显高价值的打成 A。
3. 信息还不够但值得继续看的打成 B。
4. 相关性一般的打成 C。
5. 明确不适合的直接排除，并写清原因。

### 当前页的价值不是只看基本信息

候选详情里，当前版本已经会展示和沉淀这些信息：

- 公司基础信息
- ICP 分数
- 补全状态
- 联系人覆盖情况
- AI 评估理由
- 为什么是 A / B / C 或为什么被排除

### 导入时怎么选

如果对象是“值得持续跟进的公司”，导入线索库。

如果对象本身就是“采购 / 招标 / 项目机会”，导入采购机会。

### 当前版本的一个重要边界

候选池仍然以“审核和导入”为主。
邮件序列、LinkedIn DM 等持续外联动作，当前版本已经更多迁移到线索库里继续处理。

## 4.5 线索库 `/customer/radar/prospects`

线索库是正式推进阶段的主战场。

只有进入这一页的公司，才算进入团队的正式跟进对象池。

### 当前页能做什么

这页是当前版本功能最完整的页面之一，主要能力包括：

- 查看正式线索公司
- 管理联系人
- 手动新增 / 编辑 / 删除联系人
- 手动丰富公司
- 批量丰富公司
- 生成公司背调 dossier
- 生成外联草稿
- 发送外联邮件
- 记录手动外联
- 生成 LinkedIn 草稿
- 保存外联包草稿
- 保存外联包模板
- 把外联包保存到营销内容库
- 查看外联历史
- 查看营销内容建议
- 导出 CSV

### 建议在这页的标准动作

1. 先选中一个公司。
2. 看基础信息和匹配理由。
3. 如果联系人不够，先做丰富化。
4. 联系人够用后，再生成外联包或草稿。
5. 发送后及时记录结果。
6. 如果外联材料有复用价值，保存为草稿、模板或内容库内容。

### 这页里 4 个最值得用的能力

#### 1. 丰富化公司与联系人

适合用在：

- 已导入线索，但联系人少
- 已有公司信息，但缺邮箱、电话、LinkedIn
- 想提升今日外联可执行率

#### 2. 生成 Prospect Dossier

适合在正式外联前做一轮结构化背景整理，帮助销售统一认知。

#### 3. 生成 Outreach Pack

这一套会围绕当前公司生成可复用的外联材料，通常包括：

- 开场角度
- 邮件文案
- WhatsApp 文案
- 回复 playbook
- 证据映射

#### 4. 保存到内容库

当某个行业或某类客户的外联包已经比较成熟时，可以直接保存到营销内容库，后续给营销中心复用。

### 当前页和营销中心的联动

当前版本已经支持：

- 为某个线索推荐相关营销内容
- 将外联包保存到营销内容库

这意味着现在的线索推进，不再是纯销售动作，而是可以和内容资产联动。

## 4.6 今日外联 `/customer/radar/daily`

这一页是当前版本的“日执行工作台”。

它不是简单罗列所有线索，而是把今天最值得处理的对象重新排优先级。

### 当前页会分成 3 列

- 今日电话优先
- 今日邮件优先
- 今日待补全

### 当前页适合谁用

- 销售主管
- SDR / BD
- 负责每日推进节奏的运营

### 当前页怎么用

每天进入后，建议按下面顺序处理：

1. 先处理“今日电话优先”。
2. 再处理“今日邮件优先”。
3. 最后把“今日待补全”交给继续富化。

### 当前页可直接做的动作

- 直接拨号
- 直接发邮
- 打开对应公司工作台

### 这页的意义

当前版本已经不是“发现了很多线索但不知道今天打谁”，而是开始把线索池转成可执行日清单。

## 4.7 采购机会 `/customer/radar/opportunities`

这一页用于单独管理采购和招投标类机会。

它适合和普通公司线索分开管理，因为推进逻辑不同。

### 当前页可做的事

- 查看机会列表
- 按阶段过滤
- 查看机会详情
- 更新机会阶段

### 当前支持的阶段

- `IDENTIFIED`
- `QUALIFYING`
- `PURSUING`
- `PROPOSAL`
- `NEGOTIATION`
- `WON`
- `LOST`

### 正确理解这页

普通公司型线索，主战场仍然是线索库。

只有当对象明确是一笔采购 / 招标 / 项目机会时，才更适合沉淀到这一页单独推进。

## 5. 当前版本最值得团队记住的几个规则

### 规则 1：画像先于搜索

没有画像，或者画像已经长期没更新，后面的搜索质量会明显下降。
当前流水线把画像 30 天内更新视为“新鲜”状态，超期会提醒继续同步。

### 规则 2：不要跳过候选审核

当前版本虽然能自动找客户，但仍然强调“先审核，再导入，再推进”。

建议不要直接把新发现对象一股脑当正式线索。

### 规则 3：持续搜索和即时搜索不是一回事

当前版本存在两条入口：

- 总览页“画像校正助手”：更偏向校正长期目标客户画像
- 搜索页“开始自动搜索”：更偏向即时执行一轮任务

这两个动作的使用场景不同，团队内部最好统一认知。

### 规则 4：正式外联尽量在线索库完成

当前版本把大量持续外联动作集中到了线索库。

候选池更适合做：

- 审核
- 分层
- 排除
- 导入

线索库更适合做：

- 补联系人
- 背调
- 生成外联包
- 记录跟进

### 规则 5：数据源质量要区分看待

当前系统里既有官方源，也有 AI 推断型来源。

因此：

- 高价值对象建议人工二次验证
- 采购和招标机会要特别确认时效与真实性
- 搜索页里的“健康数据源 / 官方数据源”指标要定期看

## 6. 推荐给团队的使用分工

### 销售负责人

重点看：

- 今日外联
- 线索库
- 采购机会

### 运营 / 增长负责人

重点看：

- 目标客户画像
- 自动搜索
- 候选池

### 内容 / 营销负责人

重点看：

- 线索库里的营销内容建议
- 保存到内容库的外联资产

## 7. 当前版本的边界与注意事项

为了避免团队把现阶段能力理解得过满，下面几点建议直接作为使用共识：

1. 当前主流程已经完整，但最稳定的打法仍然是“画像驱动 + 人工审核 + 线索库推进”。
2. 总览页的自然语言助手目前默认创建的是一份带定时规则的搜索配置，不是纯一次性搜索。
3. 当前自然语言助手默认启用的是 `MAPS` 和 `DIRECTORY` 两类渠道，更适合用于快速起盘。
4. 搜索页展示的数据源能力很多，但实际是否命中，仍取决于当前画像、渠道映射和数据源健康状态。
5. 候选池里已经有 AI 评估与排除反馈学习，但团队仍需要把排除原因写清楚，才能持续降低噪音。
6. 今日外联页适合日常执行，不适合作为配置入口。
7. 采购机会页适合管理项目型机会，不适合承接所有普通公司线索。
8. 当前版本是“先启动一次，后续自动按计划跑”，不是“每天都要人工点击一次”。
9. 当前版本可以自动跑，但不能直接承诺“每天固定产出 XX 个有效目标客户资料”。

## 9. 一句话总结

当前版本的获客雷达，已经不是单点搜索工具，而是一条从“画像定义”到“自动发现”再到“审核、导入、外联推进”的完整工作链。

最有效的使用方式不是把它当搜索框，而是把它当团队的日常获客操作台。
